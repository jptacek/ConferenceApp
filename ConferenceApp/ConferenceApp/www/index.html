SignalR

Getting 'Pushy' with SignalR


Track: Web/Mobile
Location: Commons
Time: 1:00 - 2:15

So you need to have a long running process in your ASP.NET web application that refreshes your page when the data is updated on the server, or perhaps even creates a Growl-like notification.

What's an ASP.NET developer to do?

You could switch to using the client side library Socket.IO and node.js on the server, but what can you do if you don't have the option of changing the technologies that you are using?

Now there is SignalR, an asynchronous signaling library for ASP.NET that is a complete client and server-side solution.

In this presentation you'll learn how you can use SignalR in your ASP.NET applications to create and manage long running asynchronouse calls that will help get one step closer to creating real-time applications.

OData Benefits

Consuming Data From Many Platforms - The Benefits of OData


Track: Micro Framework/Kinect
Location: E157
Time: 10:30-11:45

The amount of data stored today is growing at a rapid rate. However, data is only valuable if it is accessible and can be consumed by people and systems. OData is an open protocol for sharing data that is positioned to solve this problem. OData uses the standard HTTP protocol using REST principles to make data accessible and has huge industry momentum with rapid adoption growth. In this session, we will explore what OData is all about and how to expose relational and non-relational data as OData using WCF Data Services. We will then walkthrough developing apps to consume the OData feeds from multiple clients and platforms including mobile devices. Finally, we will take a look at how you can benefit from publishing your OData services to Windows Azure.

CQRS

CQRS and Event Sourcing: How it Impacts the UI


Track: Other
Location: E130
Time: 9:00 - 10:15

In this presentation we will dive into Event Sourcing. Event Sourcing is a pattern where we capture an applications state as a sequence of events. When applying this pattern alongside of CQRS you’ll notice that it has an interesting effect on how you code and design your UI. The Event Store becomes your applications source of state, while your separate query/read model is updated to reflect these changes. For this presentation we will cover the following:

  1. Overview of CQRS
  2. Create a demo application
    1. Creating an event store
    2. Model a simple domain
    3. Writing events
    4. Hydrating aggregate roots from the event store
    5. Writing event handlers to react to state changes
    6. Patterns for handling these changes in the UI

WF4 and App Fabric

Automate Your Business Processes with WF4 Services and Windows Server AppFabric


Track: Other

Time: 1:00 - 2:15

Added to the framework in .NET 3.0 and completely re-written from the ground up in .NET 4.0, XAML based WF4 workflows provide the declarative framework for building application and service logic that gives developers a higher level language for handling asynchronous, parallel tasks and other complex processing.
